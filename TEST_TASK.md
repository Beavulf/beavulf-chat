Задача: Сделать клон интерфейса чат-бота в стиле ChatGPT.

### Основные требования:

- Чат с возможностью отправки сообщений, ответы получает клиент.
- Интеграция с одной или несколькими LLM через API (OpenAI, Gemini и др.).
- Левая боковая панель со списком чатов, сохранённых в базе данных.
- Аутентификация и авторизация пользователей.
- Поддержка вставки и отправки изображений в чате.
- Анонимные пользователи могут задать до 3 бесплатных вопросов.
- Поддержка синхронизации новых чатов между разными вкладками (реального времени).
- Функция загрузки документов и получение по ним контекста.

---

### Технологический стек

**Клиент:**  
- React/Next.js (или Tanstack Start)
- Tanstack Query
- ShadcnUI + Tailwind CSS

**Сервер:**  
- Next.js REST API (или Hono/Express)

**База данных:**  
- Postgres через Supabase (или MongoDB)

**Аутентификация:**  
- Supabase (или Clerk)

**Реальное время:**  
- Supabase Realtime (или Socket.io)

**Деплой:**  
- Vercel (или Railway/Render)

---

### Архитектурные требования

- Разделять клиентский код, серверные обработчики (REST API) и базу данных.
- Все обращения к данным делать только через API-маршруты — не используйте доступ к БД напрямую в компонентах, включая серверные компоненты.
- Для работы с Supabase используйте service account через API (не применяйте публичный клиент и RLS), кроме случаев работы в реальном времени — тогда разрешён публичный клиент.

---

### Критерии для проверки:

- Грамотно спроектированные API-эндпоинты (HTTP-методы и названия).
- Оптимальная структура базы данных — без избыточности, с возможностью дальнейшего развития.
- Надёжная защита ключей API и отсутствие их утечек.
- Качественный и продуманный интерфейс: макеты, анимации, обработка загрузки, пустые состояния, адаптивность.
- Ясность и простота демонстрации.
- Присутствует инструкция по запуску проекта.

---

### Как отправить тестовое задание:

1. Опубликуйте исходный код на GitHub с файлом README (краткое описание и инструкции по запуску).
2. Запишите короткую видеопрезентацию работы вашего проекта (через Loom или другой способ записи экрана).
3. **Бонус:** Разверните приложение и предоставьте публичную ссылку (например, на Vercel или Railway).
